# PRD：A股问答对智能分析与监管合规轻量化平台

## 1. 文档信息

- 产品名称：A股问答对智能分析与监管合规轻量化平台
- 产品阶段：产品化包装 / 原型设计阶段
- 目标用途：面向实习招聘展示、产品思维训练、业务场景抽象能力展示
- 当前状态：已有模型训练与推理原型，待完善产品设计与数据飞轮机制

---

## 2. 产品背景

现有项目已经完成了一套以 A 股互动问答为核心的数据清洗、模型训练、层级分类与推理部署流程，具备以下能力：

- 对问答数据进行去噪与金融实体过滤
- 使用少量高质量 LLM 标注数据训练 Teacher 模型
- 使用 Teacher 为海量数据打软标签并蒸馏 Student 模型
- 针对直接回答与逃避回答分别构建第二层分类能力
- 已具备 FastAPI 推理原型

但从“项目”走向“产品”，仍存在明显缺口：

1. **缺少产品包装**：当前更偏模型工程，尚未完整体现业务目标、用户流程与产品结构。
2. **缺少数据飞轮**：第二层长尾类别效果不稳定，需要一套以低置信度样本为入口的复核与迭代机制。
3. **缺少可面向业务的交互层**：当前更适合技术验证，不足以完整支撑监管人员或业务人员使用。

因此，需要围绕现有项目做一次完整的轻量级产品化设计。

---

## 3. 市场机会与问题定义

### 3.1 行业问题

#### 问题 1：垂直模型缺失
目前行业内缺少专门针对复杂金融问答对进行深度解析的垂直模型。通用 NLP 模型和通用大模型在金融语境、修辞理解、董秘话术识别方面能力有限。

#### 问题 2：人工处理瓶颈
监管机构、合规部门或数据分析团队面对市场上每日新增的大量问答时，难以依赖人工方式高效处理。

#### 问题 3：长尾战术识别弱
现有模型在第一层二分类表现良好，但第二层某些逃避战术与细分类别存在明显长尾问题，需要依赖业务反馈持续优化。

### 3.2 为什么现在适合做这个产品
- 现有模型已有初步效果，可承接产品化
- 监管与合规场景对“高风险问答识别”存在天然需求
- 用低置信度复核构建数据飞轮，可以体现“产品机制服务模型迭代”的思维
- 该项目非常适合在面试中展示“技术 + 产品 + 业务理解”的复合能力

## 4. 产品定位

### 4.1 一句话定位
一个面向 **监管与合规场景** 的轻量级 A 股问答智能分析平台，通过“逃避识别 + 财务实体提取 + 低置信度复核”的方式，帮助用户高效完成海量问答筛查与持续优化。

### 4.2 产品类型
- 轻量级私有化部署工具
- AI 驱动的监管合规分析产品原型
- 从模型工程走向业务产品的包装型项目

### 4.3 目标用户
#### 主用户群 A：监管与合规端
- 交易所一线监管人员
- 证监局审查员
- 上市公司董秘办内审人员

#### 次用户群 B：量化投研端
- 量化私募研究员
- 券商资管数据分析人员
- 第三方数据加工方

### 4.4 当前优先级
当前产品优先围绕 **监管端场景** 进行设计。

## 6. 核心业务逻辑

### 6.1 两层识别机制

#### 第一层：是否逃避
输入问答对后，模型先判断该回答属于：
- 直接回答，并输出置信度。
- 逃避回答，并输出置信度。

#### 第二层：分流处理
- 如果是 **直接回答**：执行财务实体提取，包括

1. "财务表现指引"：涉及利润、营收、毛利率的具体预期或财务数据。
2. "产能与项目规划"：涉及工厂建设、产能爬坡、固定资产投资、订单数量。
3. "技术与研发进展"：涉及专利、新产品流片、临床试验进度、技术突破。
4. "资本运作与并购"：涉及增发、重组、股权激励、分红、股份回购、股东人数。
5. "合规与风险披露"：涉及诉讼、环保处罚、退市风险提示、监管问询。

- 如果是 **逃避回答**：执行逃避战术分类，包括：

1. "转移话题"：不回答核心数字或进展，而是大谈公司宏观战略、行业利好或国家政策。
2. "战略性模糊"：使用'预计'、'可能'、'不排除'等强对冲词，给出极其宽泛的区间，看似回答实则没有任何承诺。
3. "外部归因"：面对业绩下滑或项目延期，将责任完全推给疫情、地缘政治、宏观经济周期等外部因素。
4. "推迟回答"：在没有明确时间表的情况下，纯粹用'请关注后续定期报告'、'如有进展将及时披露'等合规话术挡箭牌。

### 6.2 数据飞轮机制
当模型在第一层或第二层上的置信度低于阈值时：
- 样本自动进入“待复核队列”
- 默认由人工进行确认或修改
- 可选让 Agent 针对低置信度样本给出复核建议
- 最终确认后的结果进入训练集，用于后续微调
- 基于历史语料离线发现候选词条（高频 n-gram + TF-IDF 排序），人工审核后补充词库，持续降低实体门卫漏拦截

### 6.3 产品原则
- **默认模型自动识别**，不引入 Agent 先审
- **只在低置信度场景下使用复核机制**
- **先做轻量级、刚刚好的产品架构**，避免过度设计

## 7. 核心功能列表

## 7.1 功能总览

### A. 问答分析能力
1. 单条问答分析
2. 批量 Excel 上传分析
3. 两层识别结果输出
4. 财务实体高亮
5. 逃避战术雷达图

### B. 数据飞轮能力
6. 低置信度样本自动入队
7. 人工复核
8. 可选 Agent 复核建议
9. 复核结果回流训练集

### C. 产品辅助能力
10. 审计导出
11. 模型版本记录
12. 私有化部署支持

---

## 7.2 详细功能说明

### 功能 1：单条问答分析
#### 说明
用户输入一组问答文本，系统返回完整分析结果。

#### 输入
- 公司名称
- 问答时间
- 问题文本
- 回答文本

#### 输出
- 第一层标签：直接回答 / 逃避回答
- 第一层置信度
- 第二层结果：
  - 财务实体结果
  - 或逃避战术概率分布

#### 价值
适合监管员做快速核查与示范使用。

---

### 功能 2：批量 Excel 上传分析
#### 说明
用户上传包含问答对的 Excel 文件，系统进行批量分析。

#### 目标场景
- 监管人员排查当日新增问答
- 对历史样本做批量回放与验证

#### 输出
- 批量任务状态
- 分析结果表格
- 高风险样本筛选
- 可下载导出文件

---

### 功能 3：实体与意图高亮
#### 说明
对问答文本中的财务实体进行高亮展示，并在顶部展示标签。

#### 展示内容
- 财务实体高亮
- 业务类别标签，例如：
  - 技术与研发进展
  - 财务指引
- 高风险标签，例如：
  - 疑似推迟回答
  - 疑似战略性模糊

---

### 功能 4：逃避战术雷达图
#### 说明
当样本被判定为逃避回答时，展示各类逃避战术的概率分布。

#### 作用
帮助监管人员快速理解模型判断依据与样本风险结构。

---

### 功能 5：低置信度样本队列
#### 说明
对于模型输出置信度低的样本，自动进入复核队列。

#### 触发条件
- 第一层置信度低于阈值
- 第二层置信度低于阈值
- 或命中需重点关注的长尾类别

---

### 功能 6：人工复核
#### 说明
监管员对低置信度样本进行确认、修改与备注。

#### 操作方式
- 确认当前结果
- 修改标签
- 修改战术类别
- 修改实体提取结果
- 添加备注

---

### 功能 7：可选 Agent 复核建议
#### 说明
仅在低置信度样本上，允许 Agent 给出复核建议，辅助人工判断。

#### 注意
- Agent 不进入默认主流程
- Agent 不作为先审
- Agent 仅作为低置信度样本的辅助复核工具
- Agent 建议不直接入训练集，必须由人工确认或修改后回流

---

### 功能 8：复核结果回流训练集
#### 说明
将人工确认后的结果写入训练集，用于后续微调。

#### 价值
这是产品中的关键数据飞轮能力。

## 8. 用户画像

### 8.1 监管员画像
- 目标：快速发现高风险问答样本
- 特点：不关心底层算法细节，更关注结果是否可信、是否直观、是否便于排查
- 诉求：
  - 快速浏览
  - 高风险排序
  - 结果解释清晰
  - 支持人工确认

### 8.2 投研分析人员画像
- 目标：提取可用财务信息，构建结构化数据
- 特点：更关注实体抽取结果的完整性和可导出性
- 当前优先级：低于监管端

## 9. 用户使用流程

## 9.1 单条分析流程
1. 用户进入系统
2. 输入问答文本
3. 系统执行两层识别
4. 前端展示：
   - 逃避 / 直接回答
   - 置信度
   - 雷达图或实体高亮
5. 若低置信度，则标记为“待复核”

## 9.2 批量分析流程
1. 用户上传 Excel 文件
2. 系统创建批量任务
3. 后台异步处理
4. 返回结果表格
5. 用户筛选高风险样本
6. 对低置信度样本进入复核

## 9.3 数据飞轮流程
1. 模型输出低置信度样本
2. 样本进入待复核队列
3. 人工点击“请求 Agent 建议”（可选）
4. 前端展示“模型结论 vs Agent 建议”
5. 最终标注写入训练集
6. 周期性触发微调
7. 新模型上线

> 阶段C收缩边界：不启用 Agent 全自动复核与自动入库能力，避免标签偏差被放大。

## 15. 总结

这个产品的核心不是做一个“更炫的模型展示页”，而是围绕一个真实业务问题，形成完整闭环：

- 从 **监管场景中的高风险问答筛查需求** 出发
- 以 **两层识别** 完成核心业务判断
- 通过 **低置信度复核** 构建数据飞轮
- 把已有项目升级为具备产品思维的完整方案

这使得该项目既有技术深度，也有产品与业务视角，适合作为实习招聘中的重点项目展示。
